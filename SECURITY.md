# Security Policy

## Scope

This policy applies to repositories under the blackoutsecure organization when a
repository does not define its own security policy.

## Supported Versions

Security updates are applied to the default branch. Patch releases are not maintained in this repo.

## Reporting a Vulnerability

Please report security issues privately.

1. Use the GitHub Security Advisory workflow for this repository.
2. Provide a clear description, impact, and steps to reproduce.
3. Include any relevant logs or proof-of-concept details.

## What to Include

- Affected repository and branch or commit
- Steps to reproduce the issue safely
- Expected impact and potential abuse scenarios
- Proof-of-concept details (redact sensitive data)
- Suggested mitigations (optional)

We will acknowledge reports within 3 business days and provide a timeline for remediation when available.

## Disclosure Policy

Please do not disclose vulnerabilities publicly until we confirm a fix or coordinate disclosure.
